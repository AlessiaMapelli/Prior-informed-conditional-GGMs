# Simulation Analysis

This folder contains all scripts needed to reproduce the simulation study evaluating the performance of the prior-informed covariate-dependent Gaussian Graphical Model (GGM) method.

---

## File Overview

### `ggReg_main_functions.R`, `ggReg_node_job.R`, `slurm_ggReg_node.sbatch`

These three files are identical copies from the `Computational_templates` folder. They implement the core algorithm and its HPC execution logic. Refer to that folder for documentation.

---

### `Sim_functions.R`

Contains all helper functions required to run the simulation study, including:

- **Network generation** — functions to simulate ground-truth network structures using scale-free (Barabási-Albert) or Erdős-Rényi topologies.
- **Data simulation** — functions to generate observations from a covariate-dependent GGM given a network structure and covariate configuration.
- **Prior knowledge simulation** — functions to generate perfect, noisy, or absent prior information on the network edges, with configurable sensitivity/specificity.
- **Simulation execution** - sequential computation through `run_complete_simulation()` and `generate_input_datasets_simulation()` to create the input for HPC parallel computation.
- **Evaluation** — functions to compare estimated networks against the ground truth (TPR, FPR, F1, Accuracy, magnitude preservation via Spearman correlation).
- **Result collection and processing** — `collect_and_evaluate_resuts()`, `process_simulation_results()`, `generate_summary_tables()`.

---

### `Sim_pipeline_run.R`

The main orchestration script for the simulation study. It is meant to be read and executed section by section rather than sourced as a whole, as different sections correspond to different execution modes.

**Simulation design.** The script defines the full simulation grid crossing:
- Sample sizes: 500, 1000
- Number of nodes: 50, 100
- Prior types: `perfect`, `noisy`, `none`
- Number of covariates: 1 (binary only) or 3 (2 continuous + 1 binary)

The population network follows a scale-free topology; covariate-specific networks use Erdős-Rényi graphs with edge probability 0.05. Each configuration is replicated 10 times.

**Execution modes.** The script implements three ways to run the simulation:

1. **Test run** — Runs a minimal single-configuration, single-replication version of the pipeline to verify correctness before committing to a full run.

2. **Sequential execution** — Calls `run_complete_simulation()` to run all configurations and replications one after another in a single R session. Suitable for small-scale runs or local testing.

3. **Parallel execution (HPC)** — The recommended mode for the full simulation. It works in three steps:
   - `generate_input_datasets_simulation()` pre-generates all input datasets and writes a CSV file (`input_computation_file.csv`) listing one job per row.
   - `Simulation_sequential.sh` (see below) reads that CSV and submits one SLURM array job per row, running the node-level computations in parallel on the cluster.
   - Once all jobs complete, the script collects and evaluates results via `collect_and_evaluate_resuts()`, then saves the final output.

**Post-processing.** The script also handles result processing, table generation (`generate_summary_tables()`), and figure production for the paper (Figures 1–3, saved as both PNG and SVG).

---

### `Simulation_sequential.sh`

A Bash script that submits SLURM array jobs for each simulation configuration listed in the CSV file generated by `generate_input_datasets_simulation()`.

**Usage:**
```bash
cd ./Simulation_analysis
bash Simulation_sequential.sh Simulation_results/input_computation_file.csv
```

The script reads the CSV row by row (skipping the header) and for each row submits a SLURM array job via `sbatch --array=1-${p}`, where `p` is the number of nodes. It enforces a cap of 100 concurrent jobs (`JOBS_LIMIT`), pausing submission for 2 minutes whenever the limit is reached. Each submitted job runs `slurm_ggReg_node.sbatch` with the input data path, output path, and job name read from the CSV.

The script also supports a commented-out alternative mode to process only a specific range of CSV lines (useful for resubmitting failed jobs).

## `Example_data/`

### `Example_data/Simulate_example_data.R`

This script generates the example dataset presented with the `Computational_templates` folder to illustrate the method's usage on a small, self-contained case.

It uses the same simulation infrastructure as the full study (`Sim_functions.R`) but with a minimal configuration: 10 nodes, 1000 observations, 1 binary covariate, and a perfect prior, run for a single replication. The covariate-specific network is generated with a slightly higher edge probability (0.1 vs 0.05) to ensure a non-null covariate effect at this small scale.

After generating the input data, the script extracts and renames the three objects needed by `Computational_templates`:

- `expression_data` — the simulated multivariate Gaussian observations (n × p matrix)
- `covariate_data` — the covariate matrix (n × q, with the binary covariate stored as a factor)
- `ppi_weight_matrix` — the prior weight matrix encoding edge confidence scores

These are saved both as an `.RData` file (`Example_data.RData`) and as plain-text CSV/TXT files (`expression_data.csv`, `covariate_data.csv`, `ppi_weight_matrix.txt`) directly into the `Computational_templates` folder. Intermediate simulation folders are deleted at the end, leaving only the final example files.
